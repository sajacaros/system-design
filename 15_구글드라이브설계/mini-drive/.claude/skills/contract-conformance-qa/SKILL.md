# Contract Conformance QA

## Overview

이 Skill은 검증자가 **경계면**(계약↔구현, 백엔드↔프런트)을 증거 기반으로 점검하도록 돕는다. 존재 여부만 확인하지 않고 실제 동작·보안·성능 설계를 본다. 위반은 파일·라인·테스트 결과를 근거로 지적하고 담당 빌더에게 반려한다.

## Workflow

1. 검증 대상 모듈의 계약 항목(`01-architecture/*`)을 추린다.
2. 아래 4개 체크리스트를 적용한다. 가능한 항목은 빌드·테스트로 실제 확인하고, 추정은 추정으로 표시한다.
3. 결과를 `artifacts/05-qa/{module}-conformance.md`에 남긴다.
4. 위반이 있으면 담당 빌더에게 반려 메시지를, 통합 위험이면 Orchestrator에 보고한다.

## 체크리스트

**계약 일치**
- 엔드포인트 메서드/경로/요청/응답/에러 코드가 계약과 일치
- 프런트가 계약에 없는 필드를 사용하지 않음
- WebSocket 이벤트 타입·페이로드 일치

**보안**
- JWT 만료·재발급 검증, 미인증 401
- 소유자 인가 — 남의 파일/폴더 접근 403
- 공유 토큰 expired_at 만료·비활성화 동작
- 비밀번호 해시 저장

**비기능 요구(NFR)**
- 파일 목록 조회 인덱스/페이지네이션 설계(500ms 목표)
- 실시간 알림 1초 이내 반영 경로
- 업로드 진행률·실패 재시도
- 파일 유실 0% 설계(상태머신, 버전 보존)

**시연 시나리오(PRD 11장 10단계)**
- 가입→폴더→업로드→MinIO 저장→타 브라우저 알림→다운로드→공유→타 사용자 접근→버전 생성→이전 버전 복구

## 산출물 형식 (`artifacts/05-qa/{module}-conformance.md`)

```md
## 검증 모듈:
## 판정: 통과 | 반려
## 위반 목록
| # | 항목 | 증거(파일:라인/테스트) | 심각도 | 담당 |
| --- | --- | --- | --- | --- |
## 확인된 통과 항목
-
## 사람 승인/위험 보고
-
```

## 품질 기준

- 모든 지적에 재현 가능한 증거가 있다.
- 보안·데이터 유실 위반은 심각도 높음으로 분리한다.
- 추정과 실측을 구분한다.

## 예외와 금지 행동

- 코드를 직접 수정하지 않는다(반려만).
- 근거 없이 통과/반려를 판정하지 않는다.
- 검증을 전체 빌드 종료 시점으로 미루지 않는다.
